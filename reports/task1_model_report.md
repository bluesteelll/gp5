# Задача 1. Отчёт: рекомендательная модель предсказания оценки

Полносвязная нейросеть предсказывает оценку `stars` (1–5), которую пользователь поставит
заведению, по табличным признакам пользователя и заведения. Код, обоснование и все
эксперименты — в `notebooks/Task1_rating_mlp.ipynb`; здесь сводка результатов.

## Главный результат

| | Значение |
|---|---|
| Финальная модель | ансамбль 3 сидов: InteractionMLP, ёмкость+, классиф. голова |
| **Test RMSE** | **1.0452** (bias-бейзлайн: 1.1210, выигрыш −6.8%) |
| Точное попадание в звезду | 52.2% |
| Ошибка не больше 1 звезды | 73.8% |
| Бинарно «понравится» (порог 4 звезды) | accuracy 80.1% (наивное «всем нравится» — 64.8%) |

## Данные и протокол

- Срез: Tucson + St Petersburg + Edmonton, 664 745 отзывов, 209k пользователей, 17.5k заведений.
- Временной сплит 70/15/15: train до 2019-01, val до 2020-04, test после (ковидный период,
  распределение оценок поляризовано: 53% пятёрок и 22% единиц против 43%/14% в train).
- Cold-start: 62.6% test-строк — от пользователей, которых нет в train; 20% — от новых заведений.
  Поэтому модель контентная: только признаки, без ID-эмбеддингов.
- Все скейлеры, словари и пороги фитятся только на train; предсказания клипаются в [1, 5].

## Бейзлайны (test RMSE)

| Модель | RMSE |
|---|---|
| Глобальное среднее train | 1.6534 |
| Средняя заведения | 1.4079 |
| Средняя пользователя | 1.2305 |
| Bias: user_avg + biz_avg − μ | 1.1210 |
| Ridge на 23 числовых признаках | 1.1185 |

Линейная модель не лучше суммы двух средних — выигрыш сети мог прийти только из
нелинейностей, взаимодействий и более информативных признаков.

## Что дало прирост

1. **Time-aware prior-признаки (+0.047 RMSE)** — главный рычаг. Expanding-статистики строго
   по прошлому относительно даты отзыва: средняя оценка и число отзывов пользователя/заведения
   на момент отзыва, давность последней активности. Утечки нет по построению (проверяется
   ручной сверкой в ноутбуке). Абляция: база 1.1018 → с prior 1.0548. Гео-дистанция и
   ценовая близость прироста не дали и в модель не вошли.
2. **Классификационная голова с decode-ожиданием (+0.0032 ± 0.0014, 3 сида × 3 сида)** —
   5 логитов, CrossEntropy, предсказание = Σ p(k)·k. Работает, потому что оценки бимодальны
   («либо 1, либо 5»): регрессия одним нейроном вынуждена отвечать серединой.
3. **Явное взаимодействие веток (+0.0045 при фиксированном лоссе, 3 сида)** — скалярное и
   поэлементное произведения векторов пользователя и заведения подаются в верхний MLP явно.
4. **Ёмкость и ансамбль** — cat-эмбеддинг 32, ветки 128→48, верх 256→128; усреднение
   предсказаний трёх сидов: 1.0476 → 1.0452.

## Негативные результаты (проверено и отвергнуто)

- **DCN-v2 cross-слои**: 1.0640 — хуже референса на ~30 признаках.
- **Эмбеддинг заведения**: 1.0706 при прочих равных — prior-признаки уже несут его сигнал,
  полмиллиона лишних параметров оверфитят.
- **Huber-лосс**: проигрывает MSE по RMSE (выигрывая по MAE) — выбор лосса зависит от метрики.
- **Больше данных не нужно**: кривая обучения вышла на плато (5% train → RMSE 1.113,
  100% → 1.102); узкое место — информативность признаков, а не объём.

| Сегмент | Доля | RMSE | Bias-бейзлайн |
|---|---|---|---|
| Все | 100% | 1.0452 | 1.1210 |
| Warm пользователь | 37.4% | 1.1064 | 1.1824 |

Модель выигрывает у бейзлайна на каждом сегменте, включая холодные, — подтверждение
правильности контентного подхода.

## Методология

- Все сравнения «архитектура против архитектуры» — при фиксированном лоссе и голове.
- Выбор конфигураций — только по val; test-метрики печатаются один раз, после фиксации выбора.
- Малые дельты (< 0.01 RMSE) оцениваются на трёх сидах со стандартным отклонением.
- Одиночная модель для сравнения с ансамблем — на дефолтном сиде (без подбора сида по val).
